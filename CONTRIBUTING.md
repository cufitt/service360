# Contributing to CUFitt Service360

Thank you for your interest in contributing to **CUFitt Service360**.

Service360 is an open-source, credit-union-focused service model designed for clarity, stability, and long-term adoption. Contributions are welcome, but must align with the projectâ€™s guiding principles and scope.

---

## Guiding Principles

All contributions must:
- Be **service-focused**, not transactional
- Preserve **credit-union relevance**
- Avoid introducing operational or regulatory risk
- Maintain backward compatibility where possible
- Favor clarity over complexity

Service360 is intentionally scoped. Not all feature requests will be accepted.

---

## What We Welcome

We encourage contributions in the following areas:
- Schema improvements or clarifications
- Documentation enhancements
- Diagrams and visual artifacts
- Issue reports and design discussions
- Feedback from credit unions and Microsoft partners

---

## What We Do Not Accept (Phase 1)

The following are out of scope for Phase 1:
- Core banking integrations
- Transaction processing logic
- Automated financial workflows
- Bot or Copilot implementations
- Breaking schema changes without clear justification

These areas may be revisited in future phases.

---

## How to Contribute

1. **Open an Issue First**  
   Describe the problem, use case, or proposed enhancement before submitting schema or documentation changes.

2. **Fork the Repository**  
   Create a feature branch for your proposed change.

3. **Make Focused Changes**  
   Keep pull requests small, scoped, and aligned with Service360 principles.

4. **Submit a Pull Request**  
   Clearly explain the rationale, impact, and any backward-compatibility considerations.

---

## Review Process

- All pull requests are reviewed by the **CUFitt core maintainers**,  
  led by **Sai Turlapati**, Microsoft MVP and founder of **CUFitt**.
- Feedback or requested changes may be provided prior to acceptance
- Maintainers reserve the right to decline contributions that do not align
  with the roadmap, guiding principles, or long-term product direction

---

## Code of Conduct

We expect respectful, professional, and constructive behavior from all contributors.

This project follows a simple rule:
> Be constructive, inclusive, and professional.
